# 配色番 / Haishokuban

ATSのシフト表Excelに、取引先ごとの色を自動で付ける社内ツール。

## 何をするもの

毎週カシサコさんが手で塗っているシフト表の色付けを自動化する。
**「同じ取引先で複数現場あるもの」だけに色を付けて、現場ごとに必ず違う色**にする。
隊員が現場を取り違える事故を防ぐための、安全装置としてのツール。

## 使い方（カシサコさん向け）

1. URLをブラウザで開く
2. シフト表のExcelを画面の枠にドラッグ&ドロップ
3. **全部の月に自動で色がつく**
4. プレビューしたい月を選んで結果を確認
5. **「ダウンロード」を押す**
6. ダウンロードされたExcelをLINE WORKSに貼る

## v0.8 の変更点

**色の付け方を2モードから選べるようにした**（STEP02で切り替え）。樫迫さんがどちらが見やすいか選べる。
- **会社でまとめる**：同じ会社は同系色の濃淡（中村系はぜんぶピンクの濃淡）。会社のまとまりが分かりやすい
- **現場をくっきり分ける**：同じ会社でも色をバラバラに（中村A=赤、中村A2=青、中村2=緑）。別会社で色が被るのは許容（会社名で区別できる）。同じ会社の現場の取り違え防止に最も効く

## v0.7 の変更点（大きな作り直し）

### ExcelJSを廃止、JSZip直接書き換え方式に転換
v0.6まで使っていたExcelJSは、ファイルを読み書きする際に**グラフ・数式の計算チェーン・コメント・外部リンクなどを大量に欠落**させていた（元3.2MB→2.0MB）。これがExcelを開く時の「修復しますか」エラーの原因だった。

v0.8では、元ファイルを1バイトも壊さずに保持し、**色のセルだけをXMLレベルで直接書き換える**方式（JSZip）に変更。グラフも数式も全部残る。

### 色ロジックの刷新
- **色相を取引先数で360度を等分割り当て**：固定パレットだと取引先が色数を超えたとき色相が循環して別社と衝突するため、取引先数ぶんの色相を必ず確保する
- **同じ取引先内は明度を等間隔分割**：現場が何個あっても全部違う明度になる（旧版は4段階循環で5個目以降が色重複していた）
- 結果として、**別取引先は必ず色相が違い、同じ取引先は必ず明度が違う**ので、全現場が数学的に必ずユニークな色になる

## 仕組み

- 単一の `index.html` で完結
- 外部APIなし。Excelファイルはブラウザ内で処理。サーバーには一切送信されない
- ライブラリ：JSZip（XML直接操作）、FileSaver.js
- ユーザー追加マッピングは `localStorage` で端末保存

### 色付けの内部処理
1. xlsxをJSZipで開く（ZIP構造をそのまま保持）
2. workbook.xml と rels からシート名→sheetN.xml を解決、sharedStrings をパース
3. 全月シートから現場名を集め、取引先ごとにグループ化して色プランを作成
4. styles.xml に新しい塗りつぶし（fill）とセルスタイル（cellXf）を追加
5. 各月シートのXMLで、対象範囲のセルの s 属性（スタイル番号）だけを書き換え
6. グラフ等は一切触らずZIPを再生成

## 色のロジック

- 1社で1現場 → 色なし
- 1社で複数現場 → 取引先ごとに色相を割り当て、現場ごとに明度（薄→濃）で区別
- 全現場が必ず異なる色になる

## 走査範囲

- 行12〜233、列J〜AN（10〜40列、31日分）
- 休み「-」、内勤、研修、移動日、空欄は色をクリア
